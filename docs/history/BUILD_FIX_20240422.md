# 编译错误修复记录

## 问题描述

在Linux环境编译时出现以下错误：

1. 缺少头文件导致的类型未定义
2. 缺少函数声明（malloc/free）
3. CRTSCTS 宏未定义

## 修复内容

### 1. hal_can_linux.c
添加缺失的头文件：
```c
#include <stdlib.h>      // malloc, free
#include <sys/time.h>    // struct timeval
```

### 2. hal_server_linux.c
添加缺失的头文件：
```c
#include <stdlib.h>      // malloc, free
#include <sys/time.h>    // struct timeval
```

修复 CRTSCTS 兼容性问题：
```c
#ifdef CRTSCTS
    tty.c_cflag &= ~CRTSCTS;
#endif
```

## 修复后状态

- ✅ 所有头文件已添加
- ✅ CRTSCTS 宏已条件编译
- ✅ 代码可在标准Linux环境编译

## 测试建议

```bash
# 清理后重新编译
./build.sh -c

# 本地编译
./build.sh --platform native

# 交叉编译
./build.sh --platform generic-linux
```
